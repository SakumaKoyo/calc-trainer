# 🧮 アダプティブ計算ドリルシステム (Adaptive Math Drill)

過去の解答履歴に基づいて出題確率を動的に変動させる、ルールベースの適応型学習（アダプティブ・ラーニング）エンジンを搭載した計算ドリルアプリケーションです。

単なるランダム出題や決定論的な強制復習とは異なり、「未開拓の問題へのアプローチ」と「苦手度に応じた確率の傾斜（天秤制御）」を数学的かつロジカルに両立した、中毒性と学習効率の高いドリルシステムを実現しています。

---

## 🚀 コア・アルゴリズム

### 1. 状態値（ステート）の 3値フラグデザイン
プレイヤーの解答結果は、直近3回分の履歴（最大要素数3の配列）として永続化されます。
状態値を「完全な正解（1）」と「完全な不正解（-1）」のちょうど真ん中に位置するニュートラルな値として 0 を定義することで、データ構造に繊細な意味を持たせています。

* **1 （一発正解）**: 初手で迷わず正解できた完璧な状態。
* **0 （解き直し正解）**: 初手でミスしたものの、その場で自力で解き直して正解できた状態（ミスをその場で相殺したニュートラルな状態）。
* **-1 （完全不正解）**: 初手でミスし、その場で解決できずに次の問題へ進んだ（あるいは諦めた）状態。

### 2. 精密な重み付け（ウエイト）計算式
履歴（過去3回）の中に含まれる -1 の数（absoluteWrongCount）と 0 の数（revengedWrongCount）を分断せず、地続きの1行の数式で評価します。

▼ 実装コード：
if (absoluteWrongCount + revengedWrongCount > 0) {
    return 4 + 15 * absoluteWrongCount + 4 * revengedWrongCount;
} else {
    return 1; // すべて一発正解(1)の場合はセーフモード
}

#### 📊 履歴パターン別のウエイト推移（確率の傾斜）

| 過去3回の履歴 | 未解決ミス (-1) | 解き直し (0) | ウエイト値 | 状態のニュアンスと挙動 |
| :--- | :---: | :---: | :---: | :--- |
| `[1, 1, 1]` | 0 | 0 | **1** | **克服済み（セーフモード）**：めったに出現しない。 |
| `[1, 1, 0]` | 0 | 1 | **8** | 1回初手ミスしたが直せた。警戒モードへ移行。 |
| `[1, 0, 0]` | 0 | 2 | **12** | 2回初手ミス。じわじわと出現率が上昇。 |
| `[0, 0, 0]` | 0 | 3 | **16** | 3回連続で初手ミス（泥臭く全部直した）。未解決ミス1回と同等に警戒。 |
| `[1, 1, -1]` | 1 | 0 | **19** | **完全放置ミス1回**。次回のゲームにほぼ確実に引っ張り出される。 |
| `[1, 0, -1]` | 1 | 1 | **23** | 混在パターン。直せたミスもやらかした分だけ綺麗にアドオン加算。 |
| `[0, 0, -1]` | 1 | 2 | **27** | 3回中3回ともやらかし、うち1回は未解決。高確率で最優先選出。 |
| `[1, -1, -1]` | 2 | 0 | **34** | 重度の苦手問題。 |
| `[-1, -1, -1]` | 3 | 0 | **49** | **最悪の大苦手（完全ロックオン）**。圧倒的な高確率で再出題。 |

### 3. 確率の天秤（シーソーゲーム）とゲームバランス
特定の苦手問題のウエイトを極端（100や200など）に上げすぎると、抽選プール全体が数問の苦手問題に支配され、新しい問題が供給されなくなる「スタック状態（窒息）」に陥ります。

本システムでは、初見の未解答問題をウエイト 3、完全放置の最悪問題を最大 49 に設定することで、**「苦手な問題が次回の20問に高い打率で混ざりつつも、残りの枠には新しい未解答問題が健康に流れ込んでくる」という黄金のゲームバランス**を保っています。

## 🛠️ 教育的配慮とバグ・破綻の精密ガード

### 1. 簡単すぎる「0の計算」のレアキャラ化（ウエイト 1 隔離）
九九や四則演算において、0 を含む計算（例: 7 × 0, 5 + 0）は規則さえ覚えれば一瞬で解けるため、初見時に他の通常問題と同じ確率（ウエイト 3）で頻出するとプレイヤーが退屈します。

▼ 未解答（初見）プールの判定：
if (!questionStats[key] || !questionStats[key][blankType] || questionStats[key][blankType].length === 0) {
    if (num1 === 0 || num2 === 0) return 1; // 0を含む初見問題はレア枠に隔離
    return 3; // 通常の初見問題はベースライン3
}

* **効果**：初見プレイ時は出現率を通常の3分の1に抑制し、息抜き用の「レアボーナスキャラ」として配置。
* **厳しさの両立**：もしこのボーナス問題を間違えた場合は、未解答ブロックを抜けて下のペナルティ計算（ウエイト 19〜）に移行するため、「簡単なはずなのに次回から容赦なく大激選される」という引き締まった緊張感を生みます。

### 2. 数学的「不定バグ（クイズの破綻）」のピンポイント精密狙い撃ち除去
虫食いモード（逆算ドリル）において、画面に 0 が見えている状態で特定の場所を空欄にすると、**「プレイヤーが何を打ち込んでも正解になってしまう（解が無限に存在する不定状態）」**になり、システムが崩壊します。

本システムでは、プール生成段階での一律除外（巻き添え全滅）を避け、**問題展開時に空欄の位置（b）を見ながらピンポイントで危険な組み合わせのみをガード（return）**しています。

▼ 展開エンジン内の狙い撃ちコード：
// ① 乗算：0 × [ ] = 0 (右空欄) と [ ] × 0 = 0 (左空欄) を排除
if (item.op === '*') {
    if (b === 'r' && item.num1 === 0) return;
    if (b === 'l' && item.num2 === 0) return;
}

// ② 除算：0 ÷ [ ] = 0 (右空欄、分母側) を排除
if (item.op === '/') {
    if (b === 'r' && item.num1 / item.num2 === 0) return;
}

#### ⭕ このガードによって救出される「安全な良問」
* `[ ] ÷ 3 = 0` （左空欄 ➔ 答えは 0 の1通りのみ：**出題される**）
* `0 ÷ 3 = [ ]` （答え空欄 ➔ 答えは 0 の1通りのみ：**出題される**）
* `[ ] × 3 = 0` （左空欄 ➔ 答えは 0 の1通りのみ：**出題される**）

## 💾 データ永続化設計 (Web Storage API)

アプリケーションは、サーバーレスで完全にクライアントサイド（ブラウザの localStorage）のみで完結し、高速かつ安全に動作します。

* `calc_question_stats`: 各問題キー（例: 5_*_0）および空欄タイプ（l, r, a）ごとの、直近最大3件の履歴配列（[-1, 0, 1]）を保持するコアデータベース。
* `calc_incorrect_pool`: 通常モードで間違えた問題が永続ストックされる復習用プール。復習モードで一発正解（1）を勝ち取るまで、プールから消去されない。
* `calc_training_records`: プレイごとの「日付」「モード」「誤答率」「解答速度（秒/問）」を記録し、成果を可視化するグラフ（Chart.js）のソースデータ。
